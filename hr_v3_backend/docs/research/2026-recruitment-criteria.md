# IVIS 2026 届校园招聘 — 简历筛选维度与评价标准

> 基于 2025 届 JD 更新，适用于 HR 智能筛选系统的 AI 评分配置

---

## 一、招聘概况

| 项目 | 内容 |
|------|------|
| 招聘对象 | 2026 届应届本科生/研究生 |
| 专业要求 | 计算机科学或相关专业 |
| 学历要求 | 一本及以上 |
| 招聘人数 | 日本总公司 20~30 名，上海子公司 5 名 |
| 工作地点 | 前期上海培训（~6个月）→ 日本技术培训（~3个月）→ 日本正式工作 |
| 工作内容 | 软件开发与维护、系统设计、数据库设计、模块测试 |

## 二、筛选维度

### 维度 1：技术能力（权重 40%，满分 40.00）

核心考察候选人的编程基础和技术栈匹配度。

| 级别 | 标准 | 分值范围 | 示例 |
|------|------|---------|------|
| **必须** | 掌握 Java 或 Python 至少一门 | 0.00~20.00（不满足则本维度上限 10.00） | 课程项目、实习经历中使用过 |
| **加分** | Spring / SpringCloud 框架经验 | +3.00~5.00 | 有实际项目经验，不仅限于课堂 |
| **加分** | SQL Server 或 PostgreSQL 使用经验 | +2.00~4.00 | 能进行基本的数据库设计和查询优化 |
| **加分** | 其他后端技术栈（MyBatis, Redis, Docker, Git） | 每项 +1.00~2.00 | 体现技术广度 |
| **加分** | 有完整项目经历（毕设/实习/开源） | +3.00~5.00 | 证明独立开发能力 |
| **减分** | 无任何编程语言实践经验 | -10.00~-20.00 | 仅有理论课程，无项目 |

### 维度 2：教育背景（权重 20%，满分 20.00）

| 级别 | 标准 | 分值范围 |
|------|------|---------|
| **必须** | 计算机科学、软件工程或相关专业（电子信息、数学、自动化等） | 0.00~8.00（不满足则本维度上限 5.00） |
| **必须** | 一本及以上院校（或同等水平） | 0.00~5.00（不满足则本维度上限 5.00） |
| **加分** | 985/211/双一流院校 | +2.00~4.00 |
| **加分** | 硕士/博士学历 | +1.00~3.00 |
| **加分** | GPA 优秀、奖学金获得者 | +1.00~2.00 |
| **减分** | 专业完全不相关且无转专业证明 | -5.00~-10.00 |

### 维度 3：日语能力（权重 15%，满分 15.00）

日语非硬性要求，但有基础者显著优先。

| 级别 | 标准 | 分值范围 |
|------|------|---------|
| **强加分** | JLPT N2 及以上 | +8.00~12.00 |
| **加分** | JLPT N3 / 有日语课程学习经历 | +4.00~7.00 |
| **加分** | 有日本留学/交换经历 | +3.00~5.00 |
| **中性** | 无日语基础但明确表达愿意学习 | +0.00~2.00 |
| **减分** | 明确表示不愿学习日语 | -3.00~-5.00 |

### 维度 4：跨文化适应与意愿（权重 15%，满分 15.00）

候选人需接受赴日工作安排，这是核心前提。

| 级别 | 标准 | 分值范围 |
|------|------|---------|
| **必须** | 愿意接受日本工作安排 | 0.00~5.00（不满足则本维度 0.00） |
| **加分** | 有海外留学/实习/生活经历 | +3.00~5.00 |
| **加分** | 简历中体现跨文化交流经验 | +1.00~3.00 |
| **加分** | 表达对日本文化/工作方式的兴趣 | +1.00~2.00 |
| **减分** | 简历中明确限定工作城市为国内某地 | -5.00~-8.00 |
| **强减分** | 明确不接受海外工作 | -15.00（本维度归零） |

### 维度 5：软技能与团队协作（权重 10%，满分 10.00）

| 级别 | 标准 | 分值范围 |
|------|------|---------|
| **加分** | 有团队项目经历（实习/社团/竞赛团队） | +2.00~4.00 |
| **加分** | 担任过团队负责人/项目组长 | +2.00~3.00 |
| **加分** | 沟通能力的证据（演讲、写作、协调经验） | +1.00~2.00 |
| **加分** | 编程竞赛获奖（ACM、蓝桥杯等） | +2.00~3.00 |

## 三、AI 评分系统配置

### skillConfig 推荐配置

以下配置直接对应系统中 `positions.skillConfig` 字段：

```json
{
  "must": [
    "Java",
    "Python",
    "计算机科学相关专业",
    "一本及以上学历",
    "2026届应届毕业生",
    "愿意接受日本工作安排"
  ],
  "nice": [
    "Spring",
    "SpringCloud",
    "SpringBoot",
    "MyBatis",
    "SQL Server",
    "PostgreSQL",
    "数据库设计",
    "Redis",
    "Docker",
    "Git",
    "日语",
    "JLPT",
    "N1",
    "N2",
    "N3",
    "日本留学",
    "海外经历",
    "985",
    "211",
    "双一流",
    "硕士",
    "博士",
    "研究生",
    "ACM",
    "蓝桥杯",
    "奖学金",
    "团队项目",
    "实习经历",
    "开源项目"
  ],
  "reject": [
    "不接受海外工作",
    "限定国内城市",
    "非应届",
    "无编程经验",
    "专业完全不相关"
  ]
}
```

### 评分公式

沿用现有系统公式，所有分数保留两位小数：

```
totalScore = mustScore × 0.6 + niceScore × 0.3 - rejectPenalty × 0.1
（结果保留两位小数，如 72.35）
```

| 等级 | 分数范围 | 建议动作 |
|------|---------|---------|
| A (≥80.00) | 80.00~100.00 | 核心技能全匹配 + 多项加分，优先安排面试 |
| B (≥65.00) | 65.00~79.99 | 核心技能匹配 + 部分加分，推荐面试 |
| C (≥50.00) | 50.00~64.99 | 核心技能基本匹配，备选，视名额决定 |
| D (≥35.00) | 35.00~49.99 | 核心技能部分缺失，暂不考虑 |
| F (<35.00) | 0.00~34.99 | 明显不匹配，淘汰 |

## 四、公司匹配度评价方式

除了候选人对职位的匹配，还需评估候选人与 IVIS 公司特点的契合度。

### 4.1 公司特点画像

| 维度 | IVIS 特点 |
|------|------|
| 业务模式 | 中日跨境软件开发服务 |
| 技术栈 | Java 生态为主（Spring 系），关系型数据库 |
| 工作环境 | 上海培训 → 日本长期工作，日式团队管理 |
| 团队规模 | 中小型（上海 5 人，日本 20~30 人/批） |
| 成长路径 | 技术培训 → 日语培训 → 日本项目实战 |
| 文化氛围 | 跨文化、重视团队协作、注重规范流程 |

### 4.2 公司匹配度评价维度

AI 评分提示词中应加入以下公司匹配度考量：

**高匹配度信号（在简历中寻找）：**

| 信号 | 说明 | 加分值 |
|------|------|--------|
| 日语学习经历/兴趣 | 最直接的公司匹配信号 | +5.00~10.00 |
| 海外留学/交换/实习 | 证明跨文化适应能力 | +5.00~8.00 |
| 对日本文化的兴趣 | 动漫、游学、自学日语等 | +2.00~4.00 |
| 稳定性倾向 | 非频繁跳槽型，愿意长期发展 | +2.00~3.00 |
| Java + 数据库组合 | 与公司主力技术栈高度一致 | +5.00~8.00 |
| 团队项目中的协作角色 | 公司重视团队协作而非独狼 | +2.00~4.00 |
| 培训接受度 | 愿意学习新技能，提到参加过培训 | +1.00~3.00 |
| 应届生身份 | 公司定位校招培养，非社招 | +3.00~5.00 |

**低匹配度信号（风险提示）：**

| 信号 | 说明 | 扣分值 |
|------|------|--------|
| 简历强调不愿离开某城市 | 无法赴日工作 | -10.00~-15.00 |
| 纯前端/纯算法方向 | 与公司后端为主的技术栈不匹配 | -3.00~-5.00 |
| 期望薪资远超校招水平 | 公司薪资为日本新人标准 | -3.00~-5.00 |
| 已有丰富工作经验（非应届） | 不符合校招培养模式 | -8.00~-12.00 |
| 简历无任何项目/实践 | 培训成本高，不确定性大 | -5.00~-8.00 |

### 4.3 综合评价模板

AI 评分的 `explanation` 字段应包含以下结构：

```
【技术匹配】(简要评价技术栈匹配度，维度得分 XX.XX/40.00)
【教育背景】(院校层次 + 专业相关度，维度得分 XX.XX/20.00)
【日语/跨文化】(日语水平 + 海外经历，维度得分 XX.XX/15.00)
【公司契合度】(与 IVIS 业务模式和工作方式的匹配，维度得分 XX.XX/15.00)
【软技能】(团队协作 + 沟通能力，维度得分 XX.XX/10.00)
【综合评分】totalScore = XX.XX，等级 = X
【综合建议】(一句话：推荐面试 / 备选 / 不推荐，附理由)
```

## 五、26 届 vs 25 届 JD 变化建议

基于 25 届 JD 和行业趋势，建议 26 届招聘做以下调整：

| 项目 | 25 届 | 26 届建议 | 理由 |
|------|------|------|------|
| 编程语言 | Java 或 Python | Java 或 Python，**新增 TypeScript** | 全栈趋势，公司内部已在用 |
| 框架 | Spring/SpringCloud | 保持，**新增 SpringBoot 3.x** | 市场主流已迁移到 Boot 3 |
| 数据库 | SQL Server/PostgreSQL | 保持，**强调 PostgreSQL** | 公司实际使用 PG |
| 新增技能 | 无 | **了解 AI/LLM 概念者优先** | AI 辅助开发已成趋势 |
| 日语要求 | 有基础者优先 | 保持不变 | 合理，培训体系已有 |
| 远程/混合 | 未提及 | 可注明上海培训期间的工作模式 | 26 届候选人关注此项 |
